git-nope

Guardrailed Git for Agent Workflows

git-nope is a safe Git wrapper designed to prevent agents and automation from accidentally destroying your repository. It's intentionally not Git – it's Git with training wheels.

The Problem

Agentic coding has an unfortunate tendency to… well, nope your entire repository with a misplaced git reset --hard or git checkout. What started as "hey, I can go update my code!" becomes "where the hell did everything go?"

The Solution

git-nope masks your real Git binary. When invoked as plain git, it says "nope" and refuses to operate. Agents get blocked from the dangerous stuff, while developers can still commit and stage changes using safe aliases.

Quick Install
curl -O https://github.com/your-username/git-nope/releases/latest/download/git-nope chmod +x git-nope /path/to/git-nope install # Creates symlinks in /usr/local/bin

This installs aliases like GitAdd, GitRm, GitCommit, and optionally masks your system git.

How It Works

git-nope uses a Rashomon strategy - one binary, many names:

  • GitAdd - Stages changes safely, no globbing
  • GitAddAll - Adds everything in the repo (git add -A)
  • GitAddDot - Adds current directory and below
  • GitRm - Removes files (must specify each one)
  • GitCommit - Creates commits with explicit message
Safety Features
  • No Branch Manipulation - Can't switch branches or create them
  • No History Rewriting - No reset, rebase, or otherwise dangerous commands
  • No Globs - GitRm requires explicit file names
  • Explicit Operations - Everything is intentional, nothing is accidental
⚠️ Important: This is guardrailing, not security. If an agent can bypass PATH or run /usr/bin/git directly, it can skip this. To make it robust, you need to sandbox and restrict PATH for agents.
For Agents (LLMs)

The API is dead simple - use the alphabet soup commands:

# Stage all changes GitAddAll # Remove a couple files GitRm src/old_code.py tests/broken_test.py # Commit your work GitCommit -m "Fixed the damn thing"

These match the convention of ReadFile, WriteFile, Glob, etc. Simple, explicit, hard to mess up.

Exit Codes
  • 0 - Success (including deliberate git nope)
  • 42 - Policy refusal (tried to use git for anything except git nope)
  • Other non-zero - Operational failures

Makes debugging agent workflows easy - and gives the hitchhiker's guide-philosophy of "the answer is 42, the question doesn't matter".
